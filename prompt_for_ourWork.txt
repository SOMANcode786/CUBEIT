5. SCROLL SYSTEM
==================================================

==================================================
6. IMAGE STRETCH / WAVE / ELASTIC DEFORMATION
==================================================

THIS IS CRITICAL.

During scrolling, images are NOT rigid rectangles.

Their surfaces temporarily deform.

Observed feeling:

At rest:

- images mostly return to their intended proportions
- perspective curvature remains
- no obvious wobbling

As scroll speed increases:

- images elongate very slightly in the direction of movement
- the surface bends like flexible photographic material
- vertical edges subtly bow
- parts of the image lag behind other parts
- there is slight shear
- the plane develops temporary depth
- motion has a “rubber sheet / flexible film” feeling
- distortion strength depends on scroll SPEED

When scrolling suddenly stops:

the surface must not snap instantly flat.

It should:

overshoot extremely slightly if appropriate
→ damp
→ settle smoothly back to rest.

Use REAL vertex deformation.

A useful conceptual vertex shader structure is:

vec2 centeredUv = uv - 0.5;

float verticalProfile =
    sin(uv.y * PI);

float velocity =
    clamp(uScrollVelocity, -maxVelocity, maxVelocity);

position.x +=
    velocity
    * bendStrength
    * verticalProfile;

position.z +=
    abs(velocity)
    * depthStrength
    * verticalProfile;

position.y *=
    1.0 +
    abs(velocity) * stretchStrength;

This is only the starting concept.

Improve it.

Use multiple smooth profiles if necessary so deformation is organic rather than a single obvious sine function.

For example combine:

sin(uv.y * PI)
sin(uv.y * PI * 2.0)

at very restrained amplitudes.

The top, center and bottom of the image should react slightly differently, giving the impression that the surface has inertia.

You can also introduce a tiny velocity-based UV shear in the fragment shader:

uv.x +=
    velocity *
    (uv.y - 0.5) *
    uvShearStrength;

Keep this subtle.

The effect MUST NOT:

melt faces excessively
warp typography beyond recognition
look like liquid
look like jelly constantly
look glitchy.

It should resemble a flexible, high-quality printed photograph being moved quickly through space.

==================================================
